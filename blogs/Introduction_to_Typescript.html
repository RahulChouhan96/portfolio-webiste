TypeScript is a programming language. It is statically typed and in simple terms it means every variable or object needs to have a specific data type. If you understand JavaScript, TypeScript is JavaScript + Static Type. Understand what is static and dynamic typings from below.

Static Typed: There are 2 types of languages, statically typed and dynamically typed. In statically typed, we can't change/overwrite the data type of a variable. But, in dynamically typed we can. Dynamically typed behaviour is flexible but it can lead us to unexpected errors such as a function expecting Number type and its changed to String before. 

// Dynamically Typed
let name = "Rahul";
name = 4;   // POSSIBLE

// Statically Typed
let name = "Rahul";
name = 4;   // NOT POSSIBLE

First script is written in Dynamically Typed language and you can see we can set number type value in string variable name. But in 2nd script which is Statically Typed we can't.

Why to learn TypeScript?

  1. Debugging: Debugging is process of solving errors or bugs. Debugging becomes really easy with static typed nature of TypeScript. Consider this example: 

     function getWeightInKG(weightInPound: number) {
     return weightInPound/2.2;
 }

    The above function accepts weight in pounds and converts into KG. weightInPound parameter is of type number. But what if developer passes weight string as an input (Happens quite often) then the code inside will break. So, by adding number type to parameter developer immediately get an error that it takes number as an argument and you passed string.

  2. Code Completion: When we define the type of variable or object, typeScript remembers and shows us the methods and prperties accordingly.

    In above image, you can see that by giving type number to variable num typescript shows us all methods of number.

For medium to large applications, Typescript must be used instead JavaScript.
